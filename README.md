# DSAAnalysisAndDesign
Portfolio Prep for SNHU CS-300

# What was the problem you were solving in the projects for this course?
The problem I was solving in the projects for this course was opening a file, closing a file, splitting that file and organizing the data, and being able to parse and call the right information from the file. 

# How did you approach the problem? Consider why data structures are important to understand.
I approached the problem by first starting to write the code for the menu, then working on the Binary Search Tree data structure for the sorting. Data structures are important to understand because picking the right one in the right situation is key to how your algorithm works. I chose the BST as my data structure for this reason because it automatically sorts and is much more efficient when searching than Hash Tables and Linked Lists.

# How did you overcome any roadblocks you encountered while going through the activities or project?
One large roadblock that I've ran into is wrapping my head around the logic of BST. Originally, I didn't understand how the tree sorted and inserted automatically, but by understanding that values smaller than a node go to the left and those greater go to the right, it started to make sense. 

# How has your work on this project expanded your approach to designing software and developing programs?
The work on this project taught me that jumping right into the code of the project may not be as helpful as planning what I need to do. In the project, I had to choose the data structure I wanted to use, and choosing BST was great for me because I was able to forego the sorting algorism since it's automatic.

# How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
Work on this project showed me that I should create programs that are maintainable and adaptable by creating code that is smaller and more defined as functions over large and single programs. This also helps readability because others might be able to instantly go to a function that may not be behaving as usual and fix any errors while not affect the rest of the code.
